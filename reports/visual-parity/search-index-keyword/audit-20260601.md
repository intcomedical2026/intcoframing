## Implementation Plan

1. Replace the `/index.php` route's use of `BlogListingView` with a dedicated source-compatible search page component.
2. Preserve query behavior:
   - empty `keyword=` shows the source first-page result set.
   - non-empty keyword filters products/posts while keeping the same source-compatible layout.
3. Add search-specific source CSS:
   - `.search-tool` equivalent: `50px 0`, max-width `500px`, input `50px`, border `#e5e5e5`, `22px` radius.
   - result cards: `box-shadow: 0 1px 11px 2px rgba(0,0,0,.05)`, card bottom gap `20px`, desktop image column `25%`, text column `72%`.
   - title `20px/normal`, `#666`, `500`.
   - description `16px/30px`, height `150px`, hidden overflow.
   - See More button `120x40`, `#333` background, white text.
   - mobile columns stack to `100%`, title wraps.
4. Set the header active state for `/index.php` to match the source page: Products active underline.
5. Re-capture before/after screenshots and persist Visual Ralph verdicts until score >= 90.
